# MyFood - Comprehensive AI Culinary Assistant

## 1. Project Overview
**MyFood** is a mobile application designed to function as a comprehensive culinary assistant. It solves the daily struggles of meal planning, budget management, and actively combats food waste (Zero Waste). Through an AI-powered chat, users can generate weekly meal plans, create optimized shopping lists with real-time price comparisons, and receive step-by-step cooking assistance.

**Founders:** Rodrigo Huajamaita, Lucca, and Vicenzo Giordana.
**Location:** Libertador San Martín, Entre Ríos, Argentina.
**Target Audience:** Urban professionals (ages 25-45) and young, dual-income families looking to optimize their time and household budget.

---

## 2. Technical Architecture (Stack)
For all code generated in this repository, the following technological guidelines must be strictly followed:

*   **Frontend:** React Native (iOS & Android).
*   **Backend:** Elixir with the Phoenix framework (API-only mode, `--no-html`, `--no-assets`).
*   **Database:** PostgreSQL (managed via Ecto).
*   **Real-Time Communication:** Phoenix Channels / WebSockets (crucial for streaming AI chat responses).
*   **Backend Design Pattern:** Clean Architecture. Strict separation of concerns between the Data Layer (Ecto/Repos), Business Logic (pure Phoenix Contexts), and Transport Layer (Controllers/Channels).
*   **Authentication:** JWT using the Guardian library.

---

## 3. Core Business Rules
When generating logic, adhere to the following domain rules:

1.  **Multi-tenancy (Group vs. Individual):** The system supports both individual and group (family) accounts. `MealPlans`, `Budgets`, and `ShoppingItems` belong to an `Account` (Household), not to an isolated user. All members within a group share the same visibility and data.
2.  **Budget Limits:** Every meal plan generated by the AI must be strictly constrained by the `Budget` configured for that specific `Account`.
3.  **Inventory Prioritization (Zero Waste):** The AI engine must prioritize ingredients that the user has already marked as available in their inventory to prevent expiration and waste.
4.  **Monetization (Freemium/SaaS):** Access restrictions are based on subscription tiers. Free users have limits (e.g., 3-day planning), while Premium users ($3 USD/month) have unlimited access to the cooking assistant and advanced price comparisons.

---

## 4. Core Domain Entities (Conceptual Schema)
*   **`Account`**: The primary entity that groups users. Defines the plan type (`:individual` or `:group`).
*   **`User`**: Credentials and personal data. Linked to an `Account` via an `AccountMembership`.
*   **`MealPlan`**: The weekly plan generated by the AI.
*   **`Meal`**: A specific dish assigned to a day and time (breakfast, lunch, etc.).
*   **`ShoppingItem`**: Required ingredients consolidated into a single list per `Account`.
*   **`Budget`**: The financial constraint associated with an `Account`.
*   **`Message`**: The history of interactions between the user and the AI assistant, stored under the `Account` to maintain continuous context.

## 5. AI Tone and Personality (For LLM System Prompts)
When developing the system prompts for external LLM integration (OpenAI/Gemini), the MyFood Assistant must act as a **"Personal Economist Chef"**:
*   **Tone:** Friendly, professional, warm, and highly efficient.
*   **Personality:** A blend of a nutritionist, an accountant, and an expert personal shopper who knows all the local supermarket prices.
*   **Response Objective:** Get straight to the point, maximize financial savings, minimize cooking time, and guarantee 0% food waste.
